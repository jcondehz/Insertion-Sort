#  Algoritmo de Ordenación por Inserción

En el mejor caso, si la lista ya está ordenada, el algoritmo solo la recorre una vez: O(n).
En el peor caso, con la lista en orden inverso, realiza ~n²/2 comparaciones, lo que equivale a O(n²).
En el caso promedio requiere O(n²) operaciones, por lo que solo es eficiente para listas pequeñas o casi ordenadas.
